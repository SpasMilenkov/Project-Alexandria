This guide will walk you through setting up Alexandria's development environment on your local machine. The setup uses Docker for infrastructure services and tmux for orchestrating multiple development servers with hot reload.

<Aside type="tip">
  **TL;DR**: Run `docker compose --profile garage up -d`, execute `./init-garage.sh`, configure your `.env` and `appsettings.Development.json`, then run `./run-dev.sh --s3-provider Garage`.
</Aside>

## Prerequisites

Before you begin, ensure you have the following installed:

- **Docker** (with Docker Compose)
- **tmux** — Terminal multiplexer for managing dev servers
- **.NET 10 SDK** — For backend development
- **Node.js 18+** and **pnpm** — For frontend development
- **mc (MinIO Client)** — Required only if using RustFS or Garage (optional for MinIO)

## Architecture Overview

Alexandria's dev environment runs with this architecture:

- **Docker Compose** manages infrastructure services (PostgreSQL, RabbitMQ, S3 storage)
- **tmux** orchestrates 4 development processes with hot reload:
  - Frontend (Vue + Vite)
  - API Server (ASP.NET Core)
  - Document Worker
  - Media Worker

You can choose between **three S3 providers** using Docker Compose profiles:
- **Garage** (default, lightweight, Rust-based)
- **MinIO** (S3-compatible, feature-rich)
- **RustFS** (experimental, minimal S3 implementation)

## Setup Steps

<Steps>

1. **Clone the repository**

   ```bash
   git clone https://github.com/yourusername/alexandria.git
   cd alexandria
   ```

2. **Configure environment variables**

   Copy the environment template and fill in your values:

   ```bash
   cp .env.template .env
   ```

   Edit `.env` with your preferred credentials:

   ```bash title=".env"
   # Database
   DB_PASSWORD=your_secure_password_here
   
   # MinIO (if using MinIO profile)
   MINIO_ROOT_USER=admin
   MINIO_ROOT_PASSWORD=your_secure_password_here
   
   # Garage (if using Garage profile)
   # Credentials will be generated by init-garage.sh
   
   # RustFS (if using RustFS profile)
   RUSTFS_ACCESS_KEY=dev_admin
   RUSTFS_SECRET_KEY=your_secure_password_here
   
   # RabbitMQ
   RABBITMQ_USER=dev_admin
   RABBITMQ_PASSWORD=your_secure_password_here
   RABBITMQ_VHOST=alexandria
   
   # Ports (default values, change if needed)
   POSTGRES_PORT=5432
   MINIO_API_PORT=9000
   MINIO_CONSOLE_PORT=9001
   ```

3. **Start infrastructure services**

   Choose your S3 provider and start Docker Compose:

   <Tabs>
     <TabItem label="Garage (Recommended)">
       ```bash
       # Start services with Garage profile
       docker compose -f docker-compose.dev.yml --profile garage up -d
       
       # Wait for services to be healthy (about 10-15 seconds)
       docker compose ps
       
       # Initialize Garage buckets and keys
       ./init-garage.sh
       ```
       
       The init script will output credentials. **Save these to your appsettings.Development.json** (see step 4).
     </TabItem>
     
     <TabItem label="MinIO">
       ```bash
       # Start services with MinIO profile
       docker compose -f docker-compose.dev.yml --profile minio up -d
       
       # MinIO auto-creates buckets on first access
       # No initialization script needed!
       ```
       
       MinIO Console: http://localhost:9001
       
       Login with credentials from your `.env` file.
     </TabItem>
     
     <TabItem label="RustFS">
       ```bash
       # Start services with RustFS profile
       docker compose -f docker-compose.dev.yml--profile rustfs up -d
       
       # Wait for services to be healthy
       docker compose ps
       
       # Initialize RustFS buckets
       ./init-rustfs.sh
       ```
       
       RustFS Console: http://localhost:9001
     </TabItem>
   </Tabs>

5. **Install frontend dependencies**

   ```bash
   cd Frontend
   pnpm install
   cd ..
   ```

6. **Start development servers**

   The `run-dev.sh` script launches all services in a tmux session:

   ```bash
   # Start with your chosen S3 provider
   ./run-dev.sh --s3-provider Garage
   
   # Or for MinIO/RustFS:
   # ./run-dev.sh --s3-provider MinIO
   # ./run-dev.sh --s3-provider RustFS
   ```

   This creates a tmux session with 4 windows:
   - **Frontend** — Vue dev server on http://localhost:5173
   - **API** — ASP.NET Core API on http://localhost:5000
   - **DocumentWorker** — Background worker for document processing
   - **MediaWorker** — Background worker for media processing

</Steps>

## Tmux Navigation

Once the dev environment is running, you can navigate between windows:

| Shortcut | Action |
|----------|--------|
| `Ctrl+b` then `w` | List all windows (interactive selector) |
| `Ctrl+b` then `0-3` | Switch to window by number (0=Frontend, 1=API, 2=DocWorker, 3=MediaWorker) |
| `Ctrl+b` then `d` | Detach from tmux (services keep running) |
| `tmux attach -t dev-session` | Reattach to running session |
| `Ctrl+c` in any window | Stop that service |

To stop all services:
```bash
tmux kill-session -t dev-session
docker compose down
```

## Switching S3 Providers

You can switch between S3 providers during development:

```bash
# Stop current services
tmux kill-session -t dev-session
docker compose down

# Start with different provider
docker compose --profile minio up -d

# Run initialization if needed (not needed for MinIO)
# ./init-garage.sh  OR  ./init-rustfs.sh

# Update appsettings.Development.json with new credentials

# Restart dev environment
./run-dev.sh --s3-provider MinIO
```

## Troubleshooting

### Docker containers not starting

```bash
# Check container status
docker compose ps

# View logs for specific service
docker compose logs postgres
docker compose logs garage
docker compose logs rabbitmq

# Restart a specific service
docker compose restart postgres
```

### Database migrations not applied

```bash
cd Backend
dotnet ef database update --project Data --startup-project API
```

### Frontend build errors

```bash
cd Frontend
rm -rf node_modules pnpm-lock.yaml
pnpm install
```

### S3 connection errors

Verify your S3 credentials match between:
1. Docker Compose environment variables (`.env`)
2. Initialization script output (Garage/RustFS)
3. `appsettings.Development.json` in all backend projects

Test S3 connectivity:
```bash
# For MinIO/Garage/RustFS
mc alias set test http://localhost:9000 YOUR_ACCESS_KEY YOUR_SECRET_KEY
mc ls test/
```

### "Address already in use" errors

Check if ports are already occupied:
```bash
# Check port usage
lsof -i :5432  # PostgreSQL
lsof -i :9000  # S3 API
lsof -i :5672  # RabbitMQ
lsof -i :5000  # API
lsof -i :5173  # Frontend

# Change ports in .env and docker compose.dev.yml if needed
```
